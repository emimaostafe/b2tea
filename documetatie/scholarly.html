Introducere

Scop si viziune

Tea House simuleaza o ceainarie destinata in special studentilor informaticieni, avand in vedere modul interactiv in care se pot face comenzile si se poate naviga in aplicatie. Aplicatia ofera "clientilor" un spatiu placut, estetic, potrivit pentru a servi ceai de specialitate intr-o atmosfera linistita.

Public tinta

Aplicatia este destinata in mod special studentilor informaticienti, datorita particularitatilor de contructie a unor functionalitati (navigarea in aplicatie si comandarea produselor, care se face prin construirea unei comenzi care ia forma unui URI). Acestia pot vizualiza si comanda produsele puse la dispozitie in meniul aplicatiei.

Descriere

Functiile Aplicatiei

Functionalitatile oferite de Tea House sunt urmatoarele:

  • Efectuarea de comenzi care pot fi de tip "serve in" sau "take out"
  • Vizualizarea locurilor in ceainarie
  • Selectarea unei mese, inainte de efectuarea platii unei comenzi
  • Vizualizarea meniului pus la dispozitie de catre ceainarie
  • Adaugarea in cos a produselor dorite
  • Vizualizarea notei de plata
  • Efectuare platii in maniera cash / card
  • Vizualizarea a diverse informatii de prezentare despre ceainarie
  • Vizualizarea produselor preferate de ceainarie
  • Efectuarea unei comenzi utilizand o caseta de input in format URI
  • Navigarea in aplicatie
  • Navigarea in aplicatie utilizand caseta de input in format URI

Mediul de operare

Fiind o aplicatie web, aceasta va putea rula pe orice sistem de operare cum ar fi: Android, IOS, Linux, Windows etc. In dezvoltarea aplicatiei nu au fost folosite framework-uri sub nicio forma. Au fost folosite urmatoarele tehnologii: HTML, CSS, JavaScript, PHP.

Interfata

Interfata pentru utlizator

Aplicatia ofera o interfata construita armonios, cu un design potrivit scopului cu care a fost construita. Accesarea paginilor disponibile se face dintr-o bara de navigare, care dispune de un meniu drop-down si un buton care directioneaza catre lista de produse adaugate in cos. Componentele aplicatiei, butoanele, produsele din meniu, selectarea meselor, sunt proiectate intr-o maniera comuna, intuitiva, care faciliteaza interactiunea utilizatorului cu aplicatia si facand-o o experienta placuta.

Interfete software

Baza de date folosita este o baza de date SQL, hostata in Microsoft Azure. Motivatia din spatele acestei alegeri este integrarea usoarea a limbajului de proiectare a componentei BackEnd, dar si facilitatile oferite de Azure. Baza de date SQL se integrează perfect cu Azure App Service, permitand configurarea conexiunii directe între aplicația PHP și baza de date SQL. Aceasta facilitează gestionarea și accesul la date într-un mod eficient și securizat. Apoi, baza de date SQL în Azure oferă performanță scalabilitate. Se pot ajusta ușor dimensiunea bazei de date și resursele alocate în funcție de necesități, astfel că aplicația funcționează în mod eficient chiar si cand volumul de date si traficul utilizatorilor este ridicat.

Design-ul arhitectural al aplicatiei este MVC(Model - View - Controller) pentru a modulariza aplicatia si a usura scrierea de cod. Partea de Model este asociata parii de server a aplicatiei care face interogari si preia date din baza de date. Partea de View este asociata tot a celei de server unde o parte din endpoint-uri ofera template-ul html cerut dar si alte fisiere statice cum ar fi: fisiere javascript, css, imagini, etc. In final partea de Controller este asociata partii proxy care face un request catre template-ul aplicatiei apeland un enpoint de la partea de server , preia datele necesare pentru a le atasa template-ului(oferite de microserviciul auth), le imbina folosing o functie specializata in python si returneaza pagina utilizatorului. Ca un mic exemplu, templetizand footer-url si topbar-ul separandu-le de paginile care le folosesc, modificare lor se propaga peste tot pe site deoarece la randare sunt preluate componentele si imbinate pentru a face o pagina intraga.

Frontend-ul este scris in HTML5, CSS3 si Javascript fara a folosi alte librarii. Pentru o experianta mai buna aplicatia va folosi fontul Roboto oferit de Google Fonts impreuna cu packetul de iconite.

Pentru backend se va folosi php. Arhitectura folosita este MVC.

Interfete de comunicare

Aplicatia detine enpointuri de tip Rest pe care aplicatia le foloseste prin AJAX. Formatul de trimitere/primire este JSON: /api/products /api/products/{id} /api/orders /api/orders{id}

Functionalitati

Tipul comenzii

Utilizatorul va avea posibilitatea de a alege dintre optiunile "serve in" si "take out" prin selectarea butonului respectiv. In functie de aceasta alegere, el va fi redirectionat catre o pagina urmatoare.

Vizualizare disponibilitate locuri

Daca utilizatorul doreste o comanda de tip "serve in", va fi redirectionat catre o pagina unde va putea vizualiza disponibilitatea locurilor din ceainarie. Daca sunt locuri disponibile, acesta va putea selecta masa dorita. Altfel, va putea efectua o comanda de tip "take out".

Adaugare produse in cos

Utilizatorul acceseaza pagina unde are la dispozitie meniul oferit de ceainarie ce contine pretul si detaliile despre produs. Acesta poate vizualiza si adauga produse in cos.

Vizualizare cos

Utilizatorul are posibilitatea de a verifica produsele adaugate in cos precum si suma totala pe care o are de achitat.

Efectuare plata

Utilizatorul poate sa isi achite comanda folosind plata cu cardul sau cash.

Detalii

Aplicatia pune la dispozitie utilizatorului posibilitatea de a "crea" diverite comenzi cu ajutorul unui terminal.
